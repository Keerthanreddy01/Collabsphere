# Good First Issues Guide

This file lists areas where new contributors can easily help.

## Documentation Improvements
- Fix typos anywhere in /docs
- Improve code comments in any utility file
- Add JSDoc to any function missing it

## UI Polish
- Improve any component in components/ui/
- Fix spacing or alignment issues in mobile screens

## Bug Fixes
- Any issue labeled 'good first issue' on GitHub Issues

## How to Claim an Issue
Comment "I'd like to work on this" on the issue.
The maintainer will assign it to you within 24 hours.
